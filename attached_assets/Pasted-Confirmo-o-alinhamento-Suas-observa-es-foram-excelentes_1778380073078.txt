Configure o projeto com:

Next.js 14 (App Router)
TypeScript
Tailwind CSS
Prisma ORM
Banco de dados: Neon PostgreSQL (via DATABASE_URL no .env)
Pacote de upload de imagens: Cloudinary (CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET)
Autenticação admin simples: bcrypt + cookie httpOnly
Crie o arquivo .env.example com todas as variáveis necessárias.

Crie o schema Prisma (schema.prisma) com os seguintes models:

prisma
model HeroImage {
  id        Int      @id @default(autoincrement())
  type      String
  url       String
  videoUrl  String   @default("")
  updatedAt DateTime @updatedAt
}
model GalleryImage {
  id        Int      @id @default(autoincrement())
  url       String
  alt       String   @default("")
  coupleTag String   @default("")
  location  String   @default("")
  createdAt DateTime @default(now())
}
model BlockedDate {
  id    Int      @id @default(autoincrement())
  date  DateTime
  label String   @default("Reservado")
}
model SiteConfig {
  id           Int    @id @default(autoincrement())
  weddingCount Int    @default(0)
  driverQuote  String @default("")
}
Crie também um arquivo lib/config.ts que exporta todas as variáveis de ambiente do projeto em um único lugar. Nenhuma API route deve acessar process.env diretamente — sempre via esse arquivo.

Crie um arquivo lib/storage.ts com uma função exportada uploadImage(file: File): Promise<string> que faz upload para o Cloudinary e retorna a URL. Toda lógica de upload fica isolada aqui.
